# traefik/

Single ingress
([ADR-002](https://github.com/marcoguastalli/homelab-docs/blob/main/adr/ADR-002-traefik.md),
[STD-003](https://github.com/marcoguastalli/homelab-docs/blob/main/standards/STD-003-traefik-conventions.md)).
The only stack publishing 80/443; 80 only redirects to 443.

- `config/traefik.yaml` — static: entrypoints, providers
  (`exposedByDefault: false`), ping, Prometheus metrics on :8082
  (scraped over `net_monitoring`, not published). Restart to apply.
- `config/dynamic/` — file provider, hot-reloaded: `secured` chain
  (headers + rate limit, default on every websecure router) and
  `authelia` forwardAuth; default TLS cert from `/certs`
  (= `/srv/homelab/secrets/tls`, generated by the `tls` stack).
- Dashboard: `traefik.kirito.com`, `api@internal`, behind
  `authelia@file` (two_factor) — deploy authelia first on an empty Pi.
- Docker socket mounted read-only: unavoidable for the docker provider
  ([architecture/004](https://github.com/marcoguastalli/homelab-docs/blob/main/architecture/004-security-architecture.md)).
- Stateless — no `kirito.backup` label; the cert is backed up with
  `secrets/`. No access log (SSD, STD-002 rule 13); metrics cover it.
